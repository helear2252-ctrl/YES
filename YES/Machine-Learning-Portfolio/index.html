Methodology

CRISP-DM Process Walkthrough

01

Business Understanding

定義資料分析目標與商業問題。

本專案用途

建立一個能展示機器學習、互動分析與資料視覺化能力的作品集網站。

02

Data Understanding

觀察資料分布、資料品質與異常值。

本專案用途

透過互動式線性迴歸資料，觀察樣本數、noise 與 outlier 對資料型態的影響。

03

Data Preparation

資料清理、特徵整理與資料轉換。

本專案用途

前端即時產生 X、Actual Y、Predicted Y、Error 與 Is Outlier 欄位。

04

Modeling

建立模型並觀察模型輸出。

本專案用途

使用線性迴歸概念建立回歸線，觀察斜率、截距與噪音如何改變結果。

05

Evaluation

使用指標評估模型表現。

本專案用途

使用 MAE、MSE、RMSE、R² 與 Top 10 Outlier Observations 評估預測結果。

06

Deployment

將分析成果部署成可互動使用的系統。

本專案用途

前端可部署到 GitHub Pages，未來可串接 Streamlit 後端。

遵循跨行業資料探勘標準流程，確保資料分析與模型部署的嚴謹度。

01

商業理解

定義模型商業目標與核心評估指標。

02

資料理解

搜集原始數據、進行探索性資料分析 (EDA)。

03

資料準備

資料清洗、特徵工程與離群值處理。

04

建立模型

挑選演算法、劃分訓練集並訓練模型參數。

05

模型評估

測試集驗證、誤差分析與殘差圖表檢視。

06

部署整合

串接 Python 後端、部署互動式分析面板。
